# EasyWorld 存储格式

EasyWorld 是 Ember 的默认世界存储格式:**一种格式,两个后端**,一切按地图大小自动决定加载方式。运维只需要选后端(文件 / MySQL),其余交给引擎。

## 一句话模型

```toml
# 全局默认(pumpkin.toml)
[world.chunk]
type = "easy"
backend = "file"     # 或 "mysql"(mysql 再加 url)
```

- **小地图(边界 ≤ 512×512)** → 整世界读进内存,克隆时共享内存、秒开。
- **大地图 / 无边界** → 区域惰性加载,不全部载入。
- 引擎按世界的**最大边界**判断大小,你不用选"格式"。

## 每世界配置(可选)

在某个世界的文件夹里放一个 `ember-world.toml`,只覆盖这个世界:

```toml
border   = 512            # 最大边界(格)。≤512 = 小地图;不设 = 大地图(无限)
generate = "seed"         # 生成方式:seed 按种子 | void 虚空 | ocean 海洋底
mode     = "read_write"   # read_write(默认)| read_only(不落盘,改动丢弃)
source   = "arena"        # 只读克隆:读另一个世界(arena)的数据,自己不存

[chunk]                   # 只想给这个世界换后端时才写
type = "easy"
backend = "mysql"
url = "mysql://user:pass@host:3306/ember"
```

每个键都可省;省略则用全局配置。文件写错会被**忽略并响亮报错**,不会静默改变世界行为。

### 生成方式(`generate`)

| 值 | 未生成的区块变成 |
|---|---|
| `seed`(默认) | 按世界种子正常生成地形。不动的世界照旧。 |
| `void` | 全空气(虚空)。适合自建的建筑图 / 副本。 |
| `ocean` | 基岩 + 石头 + 到海平面的水,一层空白海洋底。 |

> 默认是 `seed`,所以主世界等普通世界不受影响;虚空 / 海洋是**显式开启**的。

### 最大边界(`border`)

设了 `border` 就把世界边界钉在这个尺寸(玩家建不出界),同时 `≤512` 会被判为小地图走"整世界内存"路径。用它防止一张 512×512 的图被不小心建过界、变成大地图。

## 克隆的两种模式

- **保存克隆(持久)**:把源世界的数据复制成一个新名字并加载,可读写。
  ```
  /world clone <源> <目标>            # 默认就是保存克隆
  /world clone <源> <目标> save
  ```
- **只读克隆(临时)**:新世界读源世界的数据、改动只在内存、卸载即丢;同一个源可以同时开多个(共享内存,秒开)。
  ```
  /world clone <源> <目标> readonly
  ```
  副本就靠这个:`/dungeon start <模板>` 本质是给模板世界开一个自动命名的只读克隆。

## 两个后端

| 后端 | 数据在哪 | 跨服 |
|---|---|---|
| `file` | 世界文件夹下的 `.easy` 区域文件 | 否 |
| `mysql` | 共享 MySQL 数据库 | 是(一写多读) |

MySQL 后端支持一台读写、任意多台只读同一个世界;跨服共享要求各服的世界文件夹布局一致、`key_prefix` 相同。**注意**:跨服目前只共享方块,实体 / 传送门 POI / 玩家数据仍在各自本地。

## 命令速查

```
/world list                          # 已加载世界与在线人数
/world load / unload <名>            # 运行时加载 / 卸载(卸载先撤回默认世界)
/world tp <名>                       # 传送到该世界出生点
/world clone <源> <目标> [save|readonly]   # 保存 / 只读克隆
/world prewarm <名>                  # 把世界区域预热进内存
/world convert <名> <格式>           # 迁移未加载世界的存储格式(旧文件转 .bak)

/dungeon prewarm <模板>              # 预载副本模板进共享内存
/dungeon start <模板>                # 开一个只读副本实例并传送进去
/dungeon stop <实例>                 # 卸载实例(改动丢弃,清理目录)
/dungeon list / reload <模板>        # 查看 / 重载模板
```

权限:`ember:command.world`、`ember:command.dungeon`,默认 OP 3 级。加卸、克隆都在后台执行,不卡 tick。

## 优劣与红线

**优势**:空块修剪 + 跨块压缩(空岛 / 玩家世界省一个数量级)· 原子写崩溃不损坏 · MySQL 一写多读跨服 · 小地图秒克隆 · 加卸不卡服 · 无限世界 + 一键重置。

**红线 / 注意**:
- 数据不可丢的主世界近期仍建议 `pump` / `anvil` 观望;easy 先在可重置世界练兵。
- 默认已是 easy,但**老世界不会被自动改格式**——开档检测到磁盘是旧格式就继续用旧格式(不丢档);要真正迁移用 `/world convert <名> easy`。
- MySQL 一个库同时只能一台 `read_write`(有心跳锁保护,第二台自动降级只读)。
